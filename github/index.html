Github

- Built off of software named Git

Git

- Version control software to detect changes in text documents

Version control

- Way to save code, view history
- Similar to viewing changes in Google Docs
- Github, Bitbucket are examples of cloud version control tools

Git/Github relationship

- Save Word doc/Put in Dropbox + extra collaboration features

Github users

- Can be part of organizations
- More importantly, they own repositories

Repositories

- Store code
- Have commit histories
- Also have features like issues, Wikis, branches

Commits

- List of all the times you "saved"
- Way to track your progress

Other repository actions

- Of course you can commit to a repository
- Also push/pull, clone, fork, etc.

Basic workflow

  • Create repository for new project on Github.com and clone empty repo to your computer (this should just be an empty folder)
  • Code out part of project in that folder on your machine
  • Commit and push changes to Github using Github Desktop application (or command line if you're into that sort of thing)
  • Code more stuff, commit & push new changes, repeat
